筛法、筛法求欧拉函数、因数个数

筛法基础

给定 nnn，求出 111 ~ nnn 中所有的质数，封装成 get_prime(int n) 的数组。

暴力

不就是验证质数 nnn 遍吗？

验证质数的时间复杂度是 O(n)O(\sqrt{\smash[b]{n}})O(n​)，所以总体的时间复杂度是 O(n×n)O(\sqrt{\smash[b]{n}}\times n)O(n​×n)，非常高。

埃氏筛

质数的因数只有 111 和它本身，所以遍历到非 111 的一个数就把所有它的倍数（除了它自己）标记，遍历到后没有被标记的就是质数。

其中第 8 行 i*i是个小优化，因为比如遍历到 j=(i−1)×ij = (i - 1) \times ij=(i−1)×i，jjj 这个数就在 i−1i - 1i−1 时被标记过了。

这个时间复杂度判断较为复杂，我直接告诉答案了 O(nlog⁡log⁡n)O(n \log \log n)O(nloglogn)。

线性筛（欧拉筛）

为什么埃氏筛慢？因为它重复标记了一些数。

我们规定一个规则，一个数 mmm 只能被一个 mmm 的最小质因子 pjp_jpj​，再乘上剩下的数 iii 标记。

我们把 jjj 的循环变为遍历当前存储的质数，持续把 pj×ip_j\times ipj​×i 这个数标记。
如果 pjp_jpj​ 是 iii 的因数，跳出循环。

因为 pjp_jpj​ 是 iii 的因数，说明 pjp_jpj​ 是 iii 的最小质因子（因为质数是从大到小遍历的），那么下一个就不是 pj×ip_j \times ipj​×i 的最小质因子了，不符合上面的规则，先跳出循环。
注意：当 pjp_jpj​ 是 iii 的因数时也会标记 pj×ip_j \times ipj​×i，下一次才不标记了。

按照我们的规定，每个数只被标记了一次，所以时间复杂度为 O(n)O(n)O(n)，而且正因为如此，它叫线性筛。

筛法基础总结

我们找到了 3 种获取质数的方法，分别为：

  1. 暴力，O(n)O(\sqrt{\smash[b]{n}})O(n​)
  2. 埃氏筛，O(nlog⁡log⁡n)O(n \log \log n)O(nloglogn)
  3. 线性筛（欧拉筛），O(n)O(n)O(n)

我们一般都使用线性筛。
